# 你需要的核心思維模型

---

Claude Code 是一個在你本機環境裡跑的 AI Agent，不是聊天機器人。這個差別，決定了你應該怎麼用它。

---

## 從 ChatGPT 到 Claude Code，思維要換掉

很多人第一次用 Claude Code，腦子裡的模型還是 ChatGPT：

```
我 ── 問 ──> AI ── 答 ──> 我
```

這個模型在 Claude Code 裡是錯的。

Claude Code 的真實模型：

```
我 ── 任務 ──> AI Agent ── 讀檔案 ──> 你的 codebase
                          ── 寫檔案 ──> 你的 codebase
                          ── 執行指令 ──> 你的終端機
                          ── 查詢 ──> 網路
                          ── 回傳結果 ──> 我
```

它是個 Agent，不只是聊天機器人。這個差別決定了你怎麼用它。

---

## 概念一：Context 視窗

把 Claude Code 的「大腦」想成一個有限大小的白板。

```
┌─────────────────────────────────┐
│  Claude 的 Context 視窗          │
│                                 │
│  你的對話紀錄                    │
│  你打開的檔案                    │
│  工具執行的結果                  │
│  CLAUDE.md 的內容               │
│  ……                            │
│  [容量有限，滿了就會忘掉舊內容]   │
└─────────────────────────────────┘
```

放進 context 的資訊越精準，回答越準確。堆一堆無關的東西進去，表現反而更差。長時間的對話會讓 Claude 慢慢「忘記」早期的內容。

所以，不要把一千行的檔案全塞給它叫它「隨便看看」。要告訴它去看第 234–267 行，因為問題在那裡。

---

## 概念二：工具呼叫

Claude Code 不是直接修改你的檔案的。它的內部循環長這樣：

```
思考 → 呼叫工具 → 看結果 → 思考 → 呼叫工具 → 看結果 → ……
```

它在迭代，就像真正的工程師一樣：先讀 code、理解、寫、驗證。一個複雜的任務可能需要十幾次工具呼叫，給它時間。

---

## 概念三：權限模式

| 模式 | 行為 | 適合場景 |
|------|------|---------|
| **預設** | 寫檔案、執行指令前詢問 | 日常使用 |
| **自動批准** (`--dangerously-skip-permissions`) | 全自動，不詢問 | 信任的專案、批量操作 |

大多數時候用預設就好。快速迭代（寫測試、生成樣板）時可以暫時開自動。**永遠不要在正式環境的 server 上開自動。**

---

## 概念四：Claude Code 知道什麼

啟動時，它預設知道：
- 當前目錄的檔案結構（它會主動探索）
- `CLAUDE.md` 裡的內容（如果有的話）
- 你在對話裡說的所有事情

它不會自動知道：
- 你腦子裡的需求
- 你 code 的「背景故事」
- 你的團隊慣例
- 架構決策的歷史原因

所以你的第一個任務永遠是：給它足夠的背景。

---

## 概念五：Claude Code vs 程式碼補全工具

| 維度 | 補全工具（Copilot、Cursor） | Claude Code |
|------|--------------------------|-------------|
| 最適合 | 寫單行/單函式 | 理解和修改整個系統 |
| 上下文範圍 | 目前檔案附近 | 整個 repo + 網路 |
| 最大價值 | 速度 | 理解力 |

最佳實踐：兩個一起用。Claude Code 負責理解和設計，補全工具負責快速實作。
