# 核心心智模型

---

## TL;DR

> Claude Code 是一个在你本地环境里运行的 AI Agent。
> 它能读文件、写文件、跑命令、查网络。
> 它看到的世界 = 你给的 context + 它自己查到的东西。
> 搞懂这点，90% 的「Claude 为什么表现不稳定」就有答案了。

---

## 从 ChatGPT 到 Claude Code，思维要切换

很多人第一次用 Claude Code，脑子里还是 ChatGPT 那套模型：

```
我 ── 问 ──> AI ── 答 ──> 我
```

这个模型在 Claude Code 里是错的。

Claude Code 的真实模型：

```
我 ── 任务 ──> AI Agent ── 读文件 ──> 你的代码库
                          ── 写文件 ──> 你的代码库
                          ── 执行命令 ──> 你的终端
                          ── 查询 ──> 互联网
                          ── 返回结果 ──> 我
```

它是个 Agent，不只是聊天机器人。这个区别决定了你怎么用它。

---

## 概念一：Context 窗口

把 Claude Code 的「大脑」想成一个有限大小的白板。

放进 context 的信息越精准，回答越准确。堆一堆无关信息进去，反而表现更差。长时间对话会让 Claude 慢慢「遗忘」早期内容。

不要把一千行的文件全塞给它叫它「随便看看」。要告诉它去看第 234–267 行，因为问题在那里。

---

## 概念二：工具调用

Claude Code 不是直接修改你文件的。它的内部循环：

```
思考 → 调用工具 → 看结果 → 思考 → 调用工具 → 看结果 → ……
```

它在迭代，就像真正的程序员：先读代码、理解、写、验证。复杂任务可能需要十几次工具调用，给它时间。

---

## 概念三：权限模式

| 模式 | 行为 | 适合场景 |
|------|------|---------|
| **默认** | 写文件、执行命令前确认 | 日常使用 |
| **自动批准** (`--dangerously-skip-permissions`) | 全自动 | 信任的项目、批量操作 |

大多数时候用默认。快速迭代时可以临时开自动。**永远不要在生产环境开自动。**

---

## 概念四：它知道什么

启动时知道：
- 当前目录的文件结构
- `CLAUDE.md` 里的内容
- 你在对话里说的所有话

它不会自动知道：
- 你脑子里的需求
- 代码的「背景故事」
- 团队约定和架构决策

所以你的第一个任务永远是：给它足够的背景。

---

## 概念五：Claude Code vs 代码补全工具

| 维度 | Copilot/Cursor | Claude Code |
|------|---------------|-------------|
| 最适合 | 写单行/单函数 | 理解和修改整个系统 |
| 上下文范围 | 当前文件附近 | 整个 repo + 网络 |
| 最大价值 | 速度 | 理解力 |

最佳实践：两个一起用。Claude Code 负责理解和设计，补全工具负责快速实现。
