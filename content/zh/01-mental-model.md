# 第 1 章：你需要的核心心智模型

---

## TL;DR

> Claude Code 是一个**在你本地环境里运行的 AI Agent**。
> 它能读文件、写文件、跑命令、查网络。
> 它看到的世界 = 你给它的上下文 + 它自己查到的东西。
> 理解这一点，就理解了 90% 的"为什么 Claude 表现不稳定"的原因。

---

## 从 ChatGPT 到 Claude Code：思维上的断层

很多人第一次用 Claude Code，脑子里的模型还是 ChatGPT：

```
我 ──问──> AI ──答──> 我
```

这个模型**在 Claude Code 里是错的**。

Claude Code 的真实模型是：

```
我 ──任务──> AI Agent ──读文件──> 你的代码库
                      ──写文件──> 你的代码库
                      ──执行命令──> 你的终端
                      ──查询──> 互联网
                      ──返回结果──> 我
```

**它是一个 Agent，不只是一个聊天机器人。**

这个区别决定了你使用它的方式。

---

## 核心概念 1：上下文窗口

想象 Claude Code 的"大脑"是一张有限大小的白板。

```
┌─────────────────────────────────┐
│                                 │
│   Claude 的上下文窗口             │
│                                 │
│  你的对话历史                     │
│  你打开的文件                     │
│  工具的执行结果                   │
│  CLAUDE.md 的内容                │
│  ……                             │
│                                 │
│  [容量有限，满了就会"忘记"旧内容]  │
│                                 │
└─────────────────────────────────┘
```

**关键原则：**

- 放进上下文的信息越精准，回答越准确
- 堆越多无关信息，表现越差
- 长对话会让 Claude 慢慢"遗忘"早期的内容

**实践启示：**

不要把 1000 行的文件全塞给它，让它"随便看看"。
要告诉它去看第 234-267 行，因为问题在那里。

## 核心概念 2：工具调用（Tool Use）

Claude Code 不是直接修改你的文件的。

它会：
1. 思考需要做什么
2. 调用工具（读文件、写文件、运行命令等）
3. 看工具返回的结果
4. 继续思考
5. 调用下一个工具
6. ……直到任务完成

```
Claude 的内部循环：
THINK → ACT → OBSERVE → THINK → ACT → OBSERVE → ...
```

**这意味着：**

Claude 不是一次性"写出"你要的代码。它是在**迭代**，就像一个真正的程序员一样：先看代码，再理解，再写，再验证。

所以给它足够的时间，不要催它。一个复杂任务可能需要十几次工具调用。

---

## 核心概念 3：权限模型

Claude Code 在做任何操作之前，都会基于你的权限设置决定是否需要询问你。

三种模式：

| 模式 | 行为 | 适用场景 |
|------|------|---------|
| **Default** | 敏感操作（写文件、执行命令）需要确认 | 日常使用 |
| **Auto-approve (--dangerously-skip-permissions)** | 全自动，不询问 | 信任的项目，批量操作 |
| **Manual review** | 每一步都确认 | 关键代码，生产环境 |

**建议：**

大多数时候用默认模式。遇到需要快速迭代的场景（写测试、生成样板代码），可以临时开启自动模式。
**永远不要在生产环境的服务器上开启自动模式。**

---

## 核心概念 4：Claude Code 知道什么

启动时，Claude Code 默认知道：

1. 你当前目录的**文件结构**（它会主动探索）
2. `CLAUDE.md` 文件里写的内容（如果有的话）
3. 你在对话里说的所有话

它不会自动知道：

- 你脑子里的需求
- 你代码的"背景故事"
- 你的团队约定
- 你项目的架构决策

**所以，你的第一个任务永远是：给它足够的背景。**

---

## 核心概念 5：Claude Code vs 代码补全工具

很多人把 Claude Code 和 Copilot、Cursor 放在一起比较。这是错误的类比。

| 维度 | 代码补全工具 | Claude Code |
|------|------------|-------------|
| 适合的任务 | 写单行/单函数代码 | 理解和修改整个系统 |
| 输入方式 | 编辑器内嵌入 | 对话 + 命令行 |
| 上下文范围 | 当前文件附近 | 整个 repo + 网络 |
| 适合的工作粒度 | 微观（行/函数） | 宏观（模块/功能） |
| 最大价值 | 速度 | 理解力 |

**最佳实践：** 两者结合使用。用 Claude Code 理解和设计，用代码补全工具快速实现。
