# 序言：打破你对 AI 助手的幻想

---

## TL;DR

> 你大概一直在用 Claude Code 帮你「写代码」。
> 但它真正厉害的地方是帮你「想清楚问题、理解系统、做决策」。
> 这个区别，决定了你是工具的使用者，还是工具的主人。

---

## 凌晨一点的场景

你盯着一段 Swift 代码。

Crash 在 `viewDidLoad` 里，但怎么看都看不出问题。你打开 Claude，把代码贴进去，说：「帮我看看这段有什么问题。」

Claude 给了一段分析，听起来很有道理。你照着改，跑起来，还是崩溃。

你再问，它再答，你再改。

一个小时后，你发现问题其实是三个文件之外的一个单例，在某种竞争条件下导致的内存问题。

**Claude 没骗你。它就是不知道那三个文件的存在。**

这就是大多数人用 Claude Code 的方式：把一个盲人领进黑屋子，让他帮你找电灯开关。

---

## 真正的问题在于 context

Claude Code 只知道你告诉它的事情。你给它一个函数，它只会看这个函数。你给它整个 repo，它才能真正帮你分析。

这本书第一件事，就是让你真正搞懂「context」。掌握 context，就掌握了 Claude Code 的 80%。

---

## 三种用法

**第一种：复制粘贴型**

把代码复制进去，把答案复制出来。效率提升 20%，但总感觉差点什么。

**第二种：对话驱动型**

把 Claude Code 当成聪明的同事，会话式解决问题。效率提升 3–5 倍，但需要大量引导。

**第三种：系统思维型**

把 Claude Code 融入整个工作流，让它在正确的时间做正确的事。不只是效率，开发质量都会上一个台阶。

这本书要把你从第一种训练成第三种。

---

## 读完你会发现

- Claude Code 最强大的地方不是写代码，而是帮你理解一个陌生的 codebase
- 你的提示词质量，比模型本身更重要
- CLAUDE.md 是被最低估的功能，没有之一
- 工作流集成才是真正的乘数
