# 第 0 章：序言——打破你对 AI 助手的幻想

---

## TL;DR

> 你可能一直在用 Claude Code 帮你"写代码"。
> 但它真正能做的，是帮你"思考问题、理解系统、做出决策"。
> 这个区别，决定了你是工具的使用者，还是工具的主人。

---

## 一个真实的场景

凌晨 1 点，你盯着一段 Swift 代码。

Crash 在 `viewDidLoad` 里，但你怎么看都看不出问题。你打开 Claude，粘贴了代码，说："帮我看看这段代码有什么问题。"

Claude 给了你一段分析，看起来很有道理。你按照它说的改了，再跑，还是 Crash。

你再问，它再回答，你再改。

一个小时后，你发现问题其实是三个文件之外的一个单例在某种竞争条件下导致的内存问题。

**Claude 没骗你。它就是不知道那三个文件的存在。**

这就是大多数人使用 Claude Code 的方式：**把一个盲人领进黑屋子，让他帮你找光开关。**

---

## 真正的问题在哪里

不是 Claude 不够聪明。

问题是**上下文**（Context）。

Claude Code 只知道你告诉它的事情。你给它一个函数，它只会看这个函数。你给它整个 repo，它才能真正帮你分析。

**这本书要做的第一件事，就是让你彻底理解"上下文"这个概念。** 因为掌握上下文管理，就是掌握 Claude Code 的 80%。

---

## 三种用 Claude Code 的人

**第一种：复制粘贴型**

把代码复制进去，把答案复制出来。

结果：效率提升 20%，但总感觉差点什么。

**第二种：对话驱动型**

把 Claude Code 当成聪明的同事，会话式解决问题。

结果：效率提升 3-5 倍，但需要大量引导。

**第三种：系统思维型**

把 Claude Code 融入整个工作流，让它在正确的时间做正确的事。

结果：不只是效率，整个开发质量都上了一个台阶。

**这本书要把你从第一种训练成第三种。**

---

## 你会发现的事

读完这本书，你会意识到：

1. **Claude Code 最强大的地方不是写代码**，而是帮你理解一个陌生的代码库
2. **你的提示词质量，比模型本身更重要**
3. **CLAUDE.md 文件是被最低估的功能**，没有之一
4. **工作流集成**才是真正的乘数效应
5. **会用 Claude Code 的 iOS 开发者**，可以一个人做 3 个人的工作量

---

## 这本书不会教你的

- 如何让 Claude Code 帮你作弊（面试、考试）
- 如何用它做坏事
- 任何我自己没用过的技巧

这本书里的每一个技巧，都是经过实战验证的。不是理论，是经验。
